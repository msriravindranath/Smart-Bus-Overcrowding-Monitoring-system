System Architecture

  • Dual Ultrasonic Passenger Detection: Two ultrasonic sensors are installed at the entrance and exit of the bus door. These sensors continuously monitor the distance between the sensor and surrounding objects. When a passenger crosses the sensing zone, the system detects the trigger event and begins passenger tracking.
  • Directional Passenger Flow Detection: A time based algorithm determines the order of sensor activation. If the entrance sensor activates first followed by the exit sensor, the system identifies the passenger as entering the bus. The reverse sequence identifies passenger exit.
  • ESP32 Edge Processing Controller: The ESP32 microcontroller processes all sensor data locally. It maintains the passenger count, calculates seat availability, and manages system states while ensuring real time operation.
  • Cloud Data Storage using Google Sheets: Passenger data is transmitted via WiFi from the ESP32 to a cloud based Google Sheets database through Google Apps Script APIs. This enables centralized data storage and remote monitoring.
  • Real Time Web Dashboard: The web application retrieves bus data from the cloud and displays live passenger counts, seat availability, and bus status using interactive dashboard cards.
  • Integrated Smart Route Planner: Passengers can select origin and destination locations to view available buses along that route. The system dynamically displays bus schedules and estimated fares.
  • Live Transport Analytics: Bus occupancy levels, passenger flow events, and seat availability are continuously updated to assist transport operators in monitoring overcrowding conditions.

Future Scope

  • AI Based Passenger Recognition: Computer vision algorithms can be integrated to detect multiple passengers simultaneously and improve counting accuracy in highly crowded conditions.
  • Real Time GPS Bus Tracking: Future systems can integrate GPS modules to provide live bus location along with occupancy information.
  • Centralized Fleet Monitoring Platform: All buses in a transportation network can connect to a centralized cloud dashboard enabling city wide monitoring and analytics.
  • Passenger Mobile Application: A mobile app could allow passengers to check seat availability, bus schedules, and fares before arriving at the bus stop.
  • Smart Ticketing Integration: The system can integrate with electronic ticketing solutions to automatically validate passengers and manage seat allocation.
  • Transport Demand Prediction: Historical passenger data can be analyzed using machine learning models to predict peak demand hours and optimize bus deployment.
  • Smart City Infrastructure Integration: The system can be integrated with smart city traffic management systems to improve public transport efficiency and urban mobility.
